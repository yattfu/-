# 🧠 开发环境注意事项（Vue3 + 实时系统版）

---

## 一、核心原则（必须理解）

1. **CPU高 ≠ 有问题**
   - 编译、热更新、类型检查都会短时间拉高CPU
   - 只要能降下来，就是正常

2. **内存高 ≠ 内存不够**
   - Windows 会缓存
   - VSCode 会预加载分析项目

3. **真正的问题只有一个**
   👉 持续上涨（内存泄漏 / 监听未释放）

---

## 二、VSCode 使用规范（重点）

### 1️⃣ 必须限制文件监听

```json
"files.watcherExclude": {
  "**/node_modules/**": true,
  "**/dist/**": true,
  "**/.git/**": true
}
```

👉 否则 CPU 会一直高

---

### 2️⃣ TypeScript 内存限制

```json
"typescript.tsserver.maxTsServerMemory": 2048
```

---

### 3️⃣ ESLint 不要实时跑

```json
"eslint.run": "onSave"
```

---

### 4️⃣ 插件控制（只留必要）

必须：

- Vue（Volar）
- ESLint
- Prettier

禁止：

- 同时装 Vetur + Volar ❌

---

### 5️⃣ 项目不要多开

👉 一个 VSCode = 一个项目

---

## 三、SignalR 实时系统规范（关键）

### ✔ 页面进入连接

```js
onMounted(() => {
  realtimeStore.initRealtime();
});
```

---

### ❗ 页面销毁必须断开

```js
onBeforeUnmount(() => {
  realtimeStore.stopSignalR();
});
```

---

### ❌ 错误写法（常见坑）

```js
// 只连不关 ❌
onMounted(() => {
  realtimeStore.initRealtime();
});
```

👉 后果：

- 内存上涨
- CPU上涨
- 数据重复

---

## 四、数据处理规范（防止内存爆）

### 1️⃣ 数组必须限制长度

```js
list.push(data);

if (list.length > 100) {
  list.shift();
}
```

---

### 2️⃣ 大对象避免深响应

```js
// ❌
const data = reactive(bigObject);

// ✔
const data = shallowRef(bigObject);
```

---

### 3️⃣ 图表必须销毁

```js
onBeforeUnmount(() => {
  chart.dispose();
});
```

---

## 五、浏览器规范

### ❗不要双浏览器

- Chrome + Edge ❌
- 只留一个 ✔

---

### ❗标签页控制

👉 超过10个开始吃内存

---

## 六、开发工具规范

### ❗避免全开

避免同时运行：

- VSCode
- DBeaver
- SQL Server
- 浏览器
- 微信

---

### ✔ 推荐习惯

- 开发前关无关软件
- 数据库不用就关
- API工具按需开

---

## 七、异常判断标准

| 情况 | 是否正常 |
|------|--------|
| CPU 瞬间高 | ✅ |
| CPU 长期 50%+ | ❌ |
| 内存稳定高 | ✅ |
| 内存持续上涨 | ❌ |
| VSCode 2~3GB | ✅ |
| VSCode 5GB+ | ⚠️ |

---

## 八、你的项目特别注意点

特点：

- 实时数据（SignalR）
- 大量 SVG
- Pinia 状态管理
- 多组件

---

### ❗必须做到

- SignalR 成对（进/出）
- watcher限制
- 数据长度控制
- 图表销毁

---

## 九、一句话总结

👉 开发环境本质是：

**“实时分析系统 + 文件监听系统”**

---

## 十、终极经验

👉 大多数性能问题来自：

- 监听过多
- 实时更新过多
- 资源未释放

---

✔ 记住：

> 能关的关  
> 能限的限  
> 能释放的必须释放
